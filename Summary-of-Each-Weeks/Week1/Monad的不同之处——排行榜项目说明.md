## Monad的不同之处——排行榜项目说明

### 1.为什么排行榜需要频繁交互？

排行榜不是"一次写入、永久只读"的数据。真正的排行榜产品有大量高频写入场景：

\- **每局游戏结束**：上传分数 → 触发排名重算 → 更新名次 → 可能触发奖励发放
\- **实时对战**：比分实时变化，排名每秒都在波动
\- **赛季切换**：批量归档旧赛季 → 重置排名 → 发放赛季奖励
\- **用户操作**：查看排名证明、领取排名奖励、质押竞猜、转让排名 NFT
\- **防作弊校验**：服务端提交分数 + 链上验证，每次提交都是一次合约调用

一个日活 10 万的对战游戏，每天就是几百万次写操作。

### 2.链慢或手续费高会导致什么体验问题？



```
以太坊场景：
  用户打完一局 → 提交分数 → 等 12 秒 → 看排名变化
  手续费 $2-5 → "我打了 10 局交了 $30 gas？"
  → 用户直接关掉，回去玩 Web2 游戏
  
  更糟的：
  GameFi 项目方 → 发赛季奖励 → 5000 个地址 → gas 天价
  → 要么不发，要么只给前 10 名发 → 社区暴怒
```

慢链做排行榜就是**用拖拉机跑 F1**：功能能做，体验稀烂。

### 3.Monad 有什么优势？

\- **10,000 tps / 400ms 出块**：分数提交接近即时确认，排名刷新几乎感觉不到延迟
\- **极低手续费**：高吞吐量压低单笔成本，日活百万也不怕
\- **EVM 兼容**：Solidity 合约直接用，Foundry/Hardhat 工具链不变，迁移成本极低
\- **并行执行**：同一区块内大量用户的分数提交不互相阻塞
\- **EIP-7702**：支持 Passkey 签名，用户不需要装钱包也能上链

### 4.真的需要链上记录吗？

**纯排名展示**确实不需要区块链，MySQL + Redis 就够了。

但以下场景**必须链上**才成立：

\- **反作弊**：分数和排名规则上链，无人可篡改，主办方不能偷偷改自己的分
\- **无许可奖金池**：智能合约按排名自动分配奖金，不需要信任任何人
\- **排名即资产**：赛季排名可铸造成 NFT，可交易、可展示、可用作其他协议的凭证
\- **跨游戏身份**：你的"某游戏 Top 100 称号 NFT"可以在另一个游戏里获得特殊待遇

链上排行榜的卖点不是更快更便宜——是**无法作弊的信任 + 可组合的资产**。

**Research（300 字）**

排行榜类应用是 Monad 的理想场景，因为它天然是**高频写入 + 即时反馈**型产品。

在以太坊上构建排行榜的体验是灾难性的：一次分数提交需要 12 秒确认、2-5 美元手续费；赛季结束后给 5000 名玩家发放奖励，gas 成本可达数万美元。这迫使项目方要么半途而废，要么把排名逻辑挪到链下——但链下的排名谁都能改，信任基础就瓦解了。

Monad 从根本上改变了这个等式。**10,000 tps 吞吐量**和**400ms 出块时间**意味着：分数提交几乎实时确认，排名更新延迟从 12 秒压缩到 400ms，用户打完一局就能看到名次变化。**极低的单笔手续费**让日活百万的应用也能承担链上成本，不再需要"只给前 10 名发奖"这种妥协设计。

更重要的是 Monad 的 **EVM 完全兼容**：开发者用 Solidity 写排名合约，用 Foundry 测试部署，用 MetaMask 交互——开发体验和以太坊一致，但跑在一个快 1000 倍的引擎上。加上 EIP-7702 对 Passkey 的原生支持，用户甚至不需要安装加密钱包就能上链提交分数，这对 GameFi 产品的用户转化至关重要。

链上排行榜的价值在于**"无法作弊的排名 + 可交易的名次资产 + 自动执行的奖金分配"**——Monad 让这个愿景以可接受的成本和流畅的体验变为现实。

### Tech：高频交互 Demo 功能清单



```
项目名：LeaderboardFi — 链上排行榜原语

基础模块：
├── submitScore(gameId, score, proof)   ← 提交分数（含服务端签名防作弊）
├── getRank(gameId, player)             ← 查询当前排名
├── getLeaderboard(gameId, topN)        ← 获取前 N 名
└── claimSeasonReward(gameId)           ← 领取赛季奖励

进阶模块：
├── realtimeRanking                     ← 事件驱动的实时排名变更通知
├── seasonManager                       ← 自动赛季切换 + 历史归档
├── batchRewardDistributor              ← 批量奖励分发（按排名阶梯）
├── rankNFT                             ← 赛季排名铸造为 NFT
├── challengeSystem                     ← 向排名比你高的玩家发起挑战
└── stakingBoost                        ← 质押代币提升排名权重

集成点：
├── EIP-7702 Passkey 签名提交           ← 免钱包用户体验
├── Execution Events 实时监听           ← 前端无需轮询
├── 跨游戏排名聚合                      ← 读取其他游戏的排名 NFT
└── DeFi 可组合奖励                     ← 排名奖励自动存入生息协议
```

### Demo 工作流：

1. 用户玩一局 → 服务端签名分数 proof
2. 前端调用 `submitScore`（通过 Passkey 签名，无需 gas 感知）
3. 合约验证 proof → 写入分数 → 触发 `ScoreUpdated` 事件
4. 前端监听事件 → 实时刷新排名 UI
5. 赛季结束 → 合约自动发放奖励 → 排名 Top 100 铸造 NFT

### Ops：3 个 Meme / 社区切入点

#### 1. 「链上排名，永不 P 图」

**传播点**：截一张自己排名第 1 的图 vs 实际链上排第 500——



>  "Web2 排行榜：PS 一分钟，人人第一名。Web3 排行榜：链上查得到，P 不动。"

利用"排行榜造假"这个所有人都有共鸣的痛点，对比 Web2 截图的不可信和链上的不可篡改。适合做成对比图在 Twitter/小红书病毒传播。

#### 2.「我的排名值多少钱？— 排名 NFT 交易所」

**传播点**：把游戏排名做成可交易的 NFT，制造"某热门游戏 Top 10 账号卖了多少"的话题——



>  "昨晚冲上 Top 50 → 今早有人出价 0.5 ETH 买我的排名 NFT → 我：不卖，这是荣誉。他：再加 0.3。我：……卖了。"

把抽象的游戏成就量化成价格，制造 FOMO。适合配合排名 NFT 的 marketplace 做首发活动。

#### 3.「10 万人的奖金，一个人都没少 — Batch Reward Flex」

**传播点**：赛季结束，链上自动给 10 万个地址分发奖励，零人工、零遗漏——



>  "Web2 排行榜发奖：运营手动导出 Excel → 遗漏 → 用户开骂。Web3 排行榜发奖：智能合约自动执行 → 10 万笔秒到 → 截图发推炫耀。"

强调自动化和无信任的奖金分配，对比传统游戏"运营忘记发奖"这个高频吐槽点。适合发奖当日批量截图 + 数据可视化（多少地址、多少金额、多少秒完成）。

## 总结

排行榜作为产品方向的独特优势在于：**它既是高频交互的（需要 Monad 的性能），又是需要信任不可篡改的（需要区块链），还能自然衍生资产层（排名 NFT、奖金合约）**——三个价值维度 Monad 都能覆盖。
