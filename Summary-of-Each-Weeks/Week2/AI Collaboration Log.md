# AI Collaboration Log

## AI 协作记录：用 AI 辅助理解 Uniswap V2 并生成最小代码骨架

### 协作场景
选择 Uniswap V2 核心文档，让 AI 辅助理解恒定乘积做市商机制，
并生成最小 Solidity 合约骨架（只保留 swap / addLiquidity / removeLiquidity）。

### AI 帮了什么

| 环节             | AI 做了什么                                                  | 质量评估                           |
| ---------------- | ------------------------------------------------------------ | ---------------------------------- |
| **读文档**       | 抓取 Uniswap V2 docs + 白皮书，用中文总结核心机制（x*y=k、先转账后调 swap、Factory/Pair 架构） | ✅ 准确，比我自己翻英文文档快 10 倍 |
| **生成代码骨架** | 输出完整 Solidity 合约：SimplePair + SimpleERC20，三个核心函数 + 4 个 Foundry 测试 | ⚠️ 结构对，但有 2 个 bug            |
| **解释概念**     | 把每个函数的逻辑用注释标注，标注 [AI 辅助实现] 标记          | ✅ 帮助理解                         |

### 人类删改 / 核查了什么

| 修改                   | 问题描述                                                     | 严重程度   |
| ---------------------- | ------------------------------------------------------------ | ---------- |
| **修复 K 检查逻辑**    | AI 生成的 K 检查 `balance0*1000 * balance1*1000 >= reserve0*reserve0*1000*1000` 等价于 `balance0*balance1 >= reserve0*reserve1`，因为没有减去 0.3% 手续费，不等式恒成立，永远不触发保护 | 🔴 安全漏洞 |
| **修复 swap 执行顺序** | AI 在**发代币之前**就读余额算 amountIn，导致要取出的 output token 被误算成 input token。真实 Uniswap V2 是「先发代币 → 再读余额 → 最后算 amountIn」。这导致 amount1In 总是等于 amount1Out | 🔴 逻辑错误 |
| **修复测试用例**       | K 检查修好后，原测试用 1 wei 攻击无法触发 revert，改成 10 ether 输入请求 10 ether 输出才生效 | 🟡 测试覆盖 |

### 哪些不能交给 AI

| 不能交给 AI 的事            | 原因                                                         |
| --------------------------- | ------------------------------------------------------------ |
| **跑测试验证正确性**        | AI 生成的代码看起来自洽，但一跑就暴露两个 bug。AI 无法自己验证自己的代码能不能运行 |
| **判断 K 检查是否真的有效** | AI 写了个「做了一点事但没实际作用」的检查——人类需要理解数学才能发现 |
| **决定代码简化到什么程度**  | 去掉手续费、预言机、闪电贷是人的判断——AI 会倾向于全抄原版    |
| **最终确认安全性**          | 这是 DeFi 合约，即使是学习用途，逻辑正确性也必须人工验证     |

### 结论
这次协作中，AI 帮我省了约 80% 的时间（读文档 + 写框架代码），
但发现了 2 个会导致合约逻辑完全失效的 bug。
**AI 出的代码骨架必须跑测试验证——看起来对的代码不一定对。**
